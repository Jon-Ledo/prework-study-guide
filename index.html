HTML

  • HTML is the foundation of any web page, and can be thought of as like the sturdy foundation of a house. CSS is how the house is styled and painted, and JavaScript determines how the doors/windows open, and how the lights turn on/off
  • HTML uses elements to describe what we want tot he computer. An element is a type of content on a webpage. The following element types are commonly used to add content to pages:
    • Text Elements
    • Image Elements
    • Header Elements
    • List Elements
  • The head element contains meta information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element is an example of a semantic HTML element. In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.

CSS

  • Cascading Style Sheets (CSS), which is a programming language used by developers to define how webpage content is presented to users
  • CSS defines the colors, aesthetics, visual layout, and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all new code and prepares it to be committed
  • git commit -m: commit all your changes with a message describing exactly what it going to be added like, "added li elements and body text"
  • git pull origin main: copies code from the main branch from a repo. Useful to do before pushing, to make sure there are no surprise changes
  • git push origin 'branch-name': push your changes to the feature branch, saving it in the repo and allowing team members to view it, beginning the pull request/merging process
